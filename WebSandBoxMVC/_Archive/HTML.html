HTML Basics

HTML enables us to mark up the content to provide more meaning behind the scenes, as well as give us something to target with CSS and JavaScript. Using HTML, we can mark something as an abbreviation, create lists, display code and its output, create tables, mark content as quotes, and provide citations. Browsers may or may not provide default formatting for any particular HTML element, regardless, using CSS, we can style any element as we see fit. HTML is not used for visual effect. It is used to describe and structure your content.

Block Level vs Inline Tags

Block level tags are designed to break up the flow of content on the page with line breaks. They normally extend the entire width of their parent container and are usually designed to contain other tags (either block level or inline). Inline tags do not normally break the flow of content and instead are designed to allow other content to flow next to them. Inline tags are not designed to contain block level elements.

Common Block Level Tags

  • Heading - Range from level 1 (primary) to level 6 (lowest level)
  • Header - a container at the top to introduce a page
  • Main - a container for the primary page content
  • Footer - a container at the bottom the page, typically used to contain closing information like copyright info or lesser navigation items
  • Nav - a container intended for links, usually limited to the primary navigation
  • Article - a stand-alone content area. Typical uses are blog posts, news articles, and important self-contained topics.
  • Section - subsections that group content together and should contain a heading.
  • Aside - content that is indirectly associated with the primary content, such as secondary navigation or advertising.
  • Div - the most generic of block-level elments. It was previously the standard tag for almost everything, but has fallen out of favor with HTML5 in favor of semantic tags
  • P - Paragraph. A low-level element for text and images but it is one of the few block-level elements that is not designed to contain other block-level elements.
  • List - contains content that is broken out, typically with bullet points or numbers, and designed for readability
  • Table - designed for a speadsheet style layout. It contains tabular data laid out in rows and columns.
  • Form - Designed to collect input from the user, typically to be sent to the server for processing.

Common Inline Tags

  • img - an image to be displayed
  • a - an anchor tag typically used to create hyperlinks
  • span - a generic inline element that only exists to be targeted with CSS or JavaScript
  • strong - strong importance (bold)
  • em - emphasized (italisized)
  • button - an interface that users click on to trigger some programmed functionality
  • input - used to generate one of several types of user input controls, like text boxes, radio buttons, or checkboxes
  • code - example code
  • samp - Sample output from a computer program
  • abbr - abbreviation or acronym

Lists

Ordered Lists

Use an OL container and LI internal tags. Used for lists where the order is important.

  3. Apply jelly to the other slice of bread
  4. Put the two slices together with the peanut butter facing the jelly

Use a UL container with LI internal tags. Used for lists where the order is not important.

Semantic vs Non-Semantic Tags

Semantic tags are intended to describe the content that they contain. Examples include, strong and nav. Non-semantic tags use generalized names, such as i or div. We can still use CSS to mark up either one, but semantic tags are preferred because of their better descriptions.

Examples

This line is marked up with b for bold (non-semantic).
This line is marked up with strong to indicate that it is strongly-emphasized, important text. (semantic)

Links

Relative vs. Absolute links

These are the two most common types of links

Relative links are for files within your site. It can be defined either on the path from the current page or the path from the root folder.
Here is a relative example: Relative link to the Home page
Here is a root-relative example: Root-relative link to the Home page

Absolute links are for locations outside of your site: Centriq Training

Email links open up the default email client in order to compose and send an email to the targeted address: postmaster@centriqdev.com

Telephone links (typically used for mobile sites) are relatively new but allow phone-enabled devices to dial a number: 913-322-7000

Download links are links to non-HTML files. When clicked the default action of browsers is to start downloading the file unless the browser has the ability to read the file

Named anchor (aka bookmark) links refer to an id on the page and navigate to that location when clicked.
